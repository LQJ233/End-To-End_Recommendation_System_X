## 3. 阶段 1：前后端链路

目标：先让用户能够访问首页商品推荐，并可触发推荐刷新。
基础设施：
- Vue 3 项目搭建
- Element Plus、Pinia、Axios 集成
- 首页交互式 UI
- Spring Boot 3 Maven 项目搭建
- 商品查询接口
- 推荐刷新接口
- MySQL 商品表和用户表结构设计

预期效果：
- GET /api/v1/recommendations/home 能返回推荐商品列表
- POST /api/v1/recommendations/refresh 能触发推荐刷新
- 前端首页自动发起推荐接口请求

## 4. 阶段 2：稳定与数据链路

目标：将用户行为请求先落到 Kafka，再由服务消费写入 MySQL，并同步用户画像数据。
基础设施：
- 前端使用 IntersectionObserver 触发曝光与滚动事件上报
- 点击、曝光等事件上报前端埋点
- Nginx Lua 转发 HTTP 事件到 Kafka
- Spring Kafka Consumer 消费事件
- 服务端写入 MySQL
- Flink 实时消费 Kafka 生成 Redis 用户画像

预期效果：
- Kafka topic user_behavior_log 能持续接收用户行为数据
- Redis 用户行为列表完成更新，推荐调度可读取
- MySQL 表足够保存运营分析所需数据

## 5. 阶段 3：投入产出 MVP

目标：上线可用的召回与排序模型，保证推荐结果可见并能及时响应。
基础设施：
- Python 推荐服务
- 加载离线用户行为特征
- 加载 MMoE 排序模型
- 查询 Milvus 商品向量
- 返回 Top 500 推荐结果
- 简单结果过滤与多样性调整
- 缓存结果至 Redis

预期效果：
- 推荐刷新接口稳定可用
- 推荐服务能够产生业务可评估结果
- 推荐失败时可降级至兜底策略

## 6. 阶段 4：系统扩展与模型发布

目标：完成数据格式与模型接入产线，并发布模型服务。
基础设施：
- 数据读取与增量补偿
- Time Split 划分
- 统一特征平台整合
- Pairwise 排序接入
- 商品推荐结果接入 Milvus
- 用户 ID embedding 增量同步
- MMoE 参数变更与灰度控制
- 模型文件保存为 .pt
- TorchServe 或 Python 服务加载模型

预期效果：
- 推荐系统支持前 7 天行为特征
- 刷新结果与商品信息能接入推荐服务
- 线上运营与离线调优形成闭环

## 9. 用户画像扩展路线

阶段 1 目标：
- 通过用户画像获取准确行为特征
- JWT 验证与 token 绑定
- BCrypt 密码加密
- 前端 Axios 请求自动携带 token
- 管理端用户查询具有必要权限

阶段 2 目标：
- 密码修改
- 管理端敏感字段加密
- 管理端操作权限控制
- 用户登录行为记录

用户画像 MVP 输出：
- POST /api/v1/auth/register 可创建用户
- POST /api/v1/auth/login 可返回 token
- GET /api/v1/users/me 可获取当前用户信息
- 推荐接口校验用户 ID
